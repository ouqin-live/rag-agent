| 目标用户 | 开发者、内部问答助手、自动化评测场景 |

## 2. 背景与目标

### 2.1 当前状态
当前 `main.py` 已实现：
- 文档加载与极简分块
- 本地/离线 embedding 降级
- 内存向量检索
- OpenAI 兼容 LLM 生成 + 失败降级

但系统仍是一个**无状态脚本**：每次运行相互独立，无法记住用户，也无法判断回答好坏。

### 2.2 本里程碑目标
让 Agent 从“单次问答脚本”升级为“可长期运行、可自我评估”的 Agent：

1. **知识库能力**：支持动态增删文档、多源接入、持久化存储。
2. **记忆能力**：支持短期对话上下文 + 长期用户偏好/事实记忆。
3. **自动评估能力**：每次回答后自动打分并反馈，驱动检索与生成质量持续优化。

## 3. 范围

### 3.1 In Scope
- 知识库：多格式文档接入、持久化向量存储、增量更新
- 记忆：会话级短期记忆 + 用户级长期记忆
- 评估：RAGAS 风格指标（Faithfulness、Answer Relevance、Context Precision）+ 自定义规则
- 反馈闭环：评估结果用于召回调优与失败案例分析

### 3.2 Out of Scope
- 多 Agent 协作
- 分布式部署
- 前端 UI（保留 CLI / API 形态）
- 用户权限与多租户

## 4. 功能需求

### 4.1 知识库能力（Knowledge Base）

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| KB-01 | 多源文档加载 | P0 | 支持 txt、md、pdf、网页 URL |
| KB-02 | 可配置分块策略 | P0 | 按字符/语义/递归切分，可配置 chunk_size、overlap |
| KB-03 | 持久化向量存储 | P0 | 默认本地 SQLite / 文件存储；可选接入 Milvus/ES |
| KB-04 | 增量更新与删除 | P1 | 文档新增、更新、删除时同步更新向量库 |
| KB-05 | 混合检索 | P1 | Dense + BM25/关键词混合，支持重排序 |
| KB-06 | 元数据过滤 | P1 | 按 source、tag、date 等元数据过滤检索范围 |

### 4.2 记忆能力（Memory）

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| MEM-01 | 短期对话记忆 | P0 | 保留最近 N 轮对话，作为上下文 prompt |
| MEM-02 | 长期用户记忆 | P1 | 提取用户偏好、关键事实，持久化存储 |
| MEM-03 | 记忆检索 | P1 | 回答前检索相关长期记忆并注入 prompt |
| MEM-04 | 记忆摘要 | P2 | 长会话自动摘要，控制 token 成本 |
| MEM-05 | 记忆隐私控制 | P2 | 支持显式遗忘某条记忆 |

### 4.3 自动评估能力（Auto Evaluation）

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| EVAL-01 | Faithfulness | P0 | 生成内容是否忠于检索上下文 |
| EVAL-02 | Answer Relevance | P0 | 答案与问题的相关度 |
| EVAL-03 | Context Precision | P1 | 检索结果中相关文档占比 |
| EVAL-04 | Context Recall | P1 | 是否召回回答问题所需的全部信息 |
| EVAL-05 | 自定义规则评分 | P1 | 支持正则/关键词/毒性检测等规则 |
| EVAL-06 | 评估结果持久化 | P1 | 保存每次问答的评估分数与原因 |
| EVAL-07 | 失败案例报告 | P2 | 低分样本聚合，输出评估报告 |

## 6. 关键流程

### 6.1 问答流程
1. 接收用户问题
2. 检索长期记忆 → 注入相关事实
3. 检索知识库 → 获取 Top-K 上下文
4. 组合 prompt（系统提示 + 记忆 + 上下文 + 问题）
5. LLM 生成回答
6. 自动评估回答
7. 保存会话到短期记忆，保存评估结果
8. 返回回答 + 评估摘要

### 6.2 记忆写入流程
1. 每次对话后，提取关键信息（如偏好、事实）
2. 生成 embedding，去重后写入长期记忆库
3. 低价值/重复记忆自动合并或丢弃

### 6.3 评估反馈流程
1. 评估器对 answer / context / question 打分
2. 分数低于阈值时，标记为失败案例
3. 定期聚合失败案例，输出报告
4. 可选：根据评估结果自动调整 top_k、重排序策略

## 7. 非功能需求

| 类别 | 要求 |
|---|---|
| 离线可用 | 保留当前离线降级能力（embedding + LLM mock） |
| 可扩展 | 知识库、记忆、评估模块通过接口解耦，便于替换 |
| 可观测 | 关键步骤打印日志；评估结果可导出 JSON/CSV |
| 低成本 | 默认使用本地模型与文件存储；远程服务可选 |
| 安全性 | 长期记忆不记录敏感信息；支持显式遗忘 |

## 10. 依赖与风险

| 依赖 | 说明 |
|---|---|
| `sentence-transformers` | 已有，embedding 模型 |
| `openai` | 已有，LLM 客户端 |
| `ragas` / `langchain` / `llama-index` | 可选，用于评估与文档加载 |
| 向量数据库 | 默认本地；如需远程需环境支持 |

| 风险 | 缓解措施 |
|---|---|
| 远程 LLM/embedding 不稳定 | 保留本地降级与离线模式 |
| 长期记忆无限增长 | 摘要、去重、TTL/容量限制 |
| 自动评估打分不准 | 人机结合，保留人工标注入口 |
| 多格式文档解析复杂 | 优先支持 txt/md，pdf/网页逐步扩展 |
